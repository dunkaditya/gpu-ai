### Integration Gap: Cross-Org SSH Key Injection (HIGH)

**Location:** `internal/db/ssh_keys.go` `GetSSHKeysByIDs`

`GetSSHKeysByIDs` queries `WHERE ssh_key_id = ANY($1)` with no `AND org_id = $2` predicate. When a user provides explicit `ssh_key_ids` in `POST /api/v1/instances`, they can reference SSH key UUIDs belonging to a different organization. Those keys will be resolved and injected into the cloud-init script.

The auto-include path (`GetSSHKeysByUserID`) is correctly user-scoped. Only the explicit key ID path is affected.

**Affected requirements:** AUTH-03, SSHK-03, PRIV-07
**Fix:** Add `AND org_id = $2` to the query, or validate resolved keys belong to `req.OrgID` in `engine.Provision`.
