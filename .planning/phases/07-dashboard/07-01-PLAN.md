**Create `frontend/src/proxy.ts`** (NOT middleware.ts -- Next.js 16 uses proxy.ts):

Use a named `proxy` export (Next.js 16 convention). The function:
1. Reads `Host` header, strips port number
2. Reads `searchParams` for `site` query param
3. If hostname === `cloud.gpu.ai` OR searchParams.get('site') === 'cloud' → rewrite to `/(cloud)${pathname}`
4. Otherwise → rewrite to `/(marketing)${pathname}`
5. Uses `NextResponse.rewrite(url)` for both cases

Export a `config` object with matcher that excludes: `api`, `_next/static`, `_next/image`, `favicon.ico`, `fonts`, and common static file extensions (svg, png, jpg, jpeg, gif, webp, woff2, ico).

Pattern from research:
```typescript
import { NextResponse, type NextRequest } from 'next/server'

export function proxy(request: NextRequest) {
  const hostname = request.headers.get('host')?.split(':')[0] ?? ''
  const { searchParams, pathname } = request.nextUrl
  const isCloud = hostname === 'cloud.gpu.ai' || searchParams.get('site') === 'cloud'

  const url = request.nextUrl.clone()
  url.pathname = isCloud ? `/(cloud)${pathname}` : `/(marketing)${pathname}`
  return NextResponse.rewrite(url)
}

export const config = {
  matcher: [
    '/((?!api|_next/static|_next/image|favicon.ico|fonts|.*\\.(?:svg|png|jpg|jpeg|gif|webp|woff2?|ico)$).*)',
  ],
}
```
