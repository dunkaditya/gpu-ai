**Update `frontend/src/proxy.ts`** to compose Clerk auth with hostname routing:

Replace the current `proxy` function with `clerkMiddleware` wrapping:
```typescript
import { clerkMiddleware, createRouteMatcher } from '@clerk/nextjs/server'
import { NextResponse } from 'next/server'

const isPublicRoute = createRouteMatcher([
  '/',
  '/sign-in(.*)',
  '/sign-up(.*)',
])

export default clerkMiddleware(async (auth, req) => {
  const hostname = req.headers.get('host')?.split(':')[0] ?? ''
  const { searchParams, pathname } = req.nextUrl
  const isCloud = hostname === 'cloud.gpu.ai' || searchParams.get('site') === 'cloud'

  // Protect cloud routes (require authentication)
  if (isCloud && !isPublicRoute(req)) {
    await auth.protect()
  }

  // Rewrite to appropriate route group
  const url = req.nextUrl.clone()
  if (isCloud) {
    url.pathname = `/(cloud)${pathname}`
  } else {
    url.pathname = `/(marketing)${pathname}`
  }
  return NextResponse.rewrite(url)
})

export const config = {
  matcher: [
    '/((?!api|_next/static|_next/image|favicon.ico|fonts|.*\\.(?:svg|png|jpg|jpeg|gif|webp|woff2?|ico)$).*)',
  ],
}
```

Key changes from Plan 01:
- `export function proxy` replaced with `export default clerkMiddleware(...)`
- `auth.protect()` called for cloud routes that aren't public
- Sign-in/sign-up paths are marked as public routes
- Marketing routes remain fully public (no auth.protect)

**NOTE:** With `clerkMiddleware` as default export, Next.js 16 still picks it up. The `proxy.ts` file with a default export works alongside the named `proxy` export convention. If Next.js 16 strictly requires named `proxy` export, wrap as: `export const proxy = clerkMiddleware(...)`. Test which works -- try default export first (Clerk docs recommend it), fall back to named export if build fails.
