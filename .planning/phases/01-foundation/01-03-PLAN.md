   - Create pool via `pgxpool.NewWithConfig(ctx, config)`

6. Implement a **standalone retry helper** at package level (not on Pool, since it is used before Pool exists):
   ```go
   func ConnectWithRetry(ctx context.Context, name string, maxRetries int, connect func(ctx context.Context) error) error
   ```
   - Exponential backoff: delay = min(1s * 2^attempt, 30s)
   - Use `select` with `ctx.Done()` and `time.After(delay)` to respect context cancellation
   - Log each retry attempt via `slog.Warn("connection failed, retrying", "service", name, "attempt", i+1, "delay", delay, "error", err)`
   - On final failure: return `fmt.Errorf("failed to connect to %s after %d attempts: %w", name, maxRetries, err)`

The retry helper is generic so main.go can use it for BOTH Postgres and Redis connections.

Imports: `context`, `fmt`, `log/slog`, `time`, `github.com/jackc/pgx/v5/pgxpool`

Note: Redis client creation will happen in main.go directly using `redis.ParseURL()` + `redis.NewClient()` since go-redis does not need a wrapper struct for Phase 1. The retry helper from this package will be used for the Redis ping.
