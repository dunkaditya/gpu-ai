**Create `internal/api/handlers_ssh_keys.go`:**

Add SSH key request/response types:
```go
type AddSSHKeyRequest struct {
    Name      string `json:"name"`
    PublicKey string `json:"public_key"`
}

type SSHKeyResponse struct {
    ID          string `json:"id"`
    Name        string `json:"name"`
    Fingerprint string `json:"fingerprint"`
    CreatedAt   string `json:"created_at"`
}
```

**`handleCreateSSHKey`** (POST /api/v1/ssh-keys):
1. Decode JSON body into `AddSSHKeyRequest`. Validate: name non-empty (max 100 chars), public_key non-empty.
2. Parse and validate key using `golang.org/x/crypto/ssh`: `ssh.ParseAuthorizedKey([]byte(req.PublicKey))`. If error, return 400 with problem detail "invalid_ssh_key".
3. Check key type: `pubKey.Type()` must be one of `ssh-rsa`, `ssh-ed25519`, `ecdsa-sha2-nistp256`, `ecdsa-sha2-nistp384`, `ecdsa-sha2-nistp521`. Reject anything else (including `ssh-dss` for DSA) with 400 "unsupported_key_type".
4. Compute fingerprint: `ssh.FingerprintSHA256(pubKey)` — already returns `SHA256:base64...` format.
5. Check org key count: call `s.db.CountSSHKeysByOrg(ctx, orgID)`. If >= 50, return 422 with problem detail "key_limit_exceeded", detail "Maximum 50 SSH keys per organization".
6. Build `db.SSHKey{UserID: claims.UserID, OrgID: orgID, Name: req.Name, PublicKey: req.PublicKey (trimmed), Fingerprint: fp}`. Call `s.db.CreateSSHKey`. If `db.ErrDuplicateKey`, return 409 with "duplicate_key".
7. Return 201 with `SSHKeyResponse`.

**`handleListSSHKeys`** (GET /api/v1/ssh-keys):
Call `s.db.ListSSHKeysByOrg(ctx, orgID)`. Return 200 with `{"ssh_keys": [...]}` array. No pagination needed for Phase 5 (max 50 keys per org fits in single response).

**`handleDeleteSSHKey`** (DELETE /api/v1/ssh-keys/{id}):
Extract `{id}` from path. Call `s.db.DeleteSSHKey(ctx, id, orgID)`. If not found, return 404. Return 204 on success.

**Update `internal/provision/engine.go` — smart SSH key default:**
In `Provision()`, replace the block that calls `GetSSHKeysByIDs` and returns `ErrSSHKeysNotFound` if empty. New logic:
```go
var sshKeys []db.SSHKey
if len(req.SSHKeyIDs) > 0 {
    // Explicit key IDs provided — look them up.
    sshKeys, err = e.db.GetSSHKeysByIDs(ctx, req.SSHKeyIDs)
    if err != nil {
        return nil, fmt.Errorf("provision: look up ssh keys: %w", err)
    }
} else {
    // No key IDs provided — auto-include all of user's keys.
    sshKeys, err = e.db.GetSSHKeysByUserID(ctx, req.UserID)
    if err != nil {
        return nil, fmt.Errorf("provision: look up user ssh keys: %w", err)
    }
}
if len(sshKeys) == 0 {
    return nil, ErrSSHKeysNotFound // "at least one SSH key required"
}
```

The handler in `handlers.go` should pass the `UserID` through to `ProvisionRequest` (it already does per the existing struct). Ensure the create handler passes `claims.UserID` as `UserID` in `ProvisionRequest`.

Import `golang.org/x/crypto/ssh` in handlers_ssh_keys.go — already in go.mod as `golang.org/x/crypto v0.43.0`.
