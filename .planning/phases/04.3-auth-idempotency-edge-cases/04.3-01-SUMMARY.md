- Pattern 2 (Eager Creation) chosen over Pattern 1 (Pass-Through) for idempotency middleware -- ensures idempotency key is always stored, even on first request
- EnsureOrg extracted as standalone function rather than duplicating org upsert logic in middleware
- NULLIF used to convert empty email strings to NULL in upsert queries (cleaner than storing empty strings)
- RETURNING user_id used on user upsert to get internal UUID in a single query round-trip
