Fix two structural bugs in auth and idempotency that block multi-user organizations and first-request idempotency.

Purpose: AUTH-03 requires multiple users per org. The email UNIQUE constraint blocks this. API-11 requires idempotency to work on all requests, but the middleware fails with 500 for a new org's first request because the org doesn't exist yet in the DB. Additionally, fix the user_id FK mismatch where the Clerk user ID (a string) was being stored instead of the internal user UUID.

Output: Migration v4, refactored EnsureOrg/EnsureOrgAndUser, fixed idempotency middleware, fixed handler user_id.
</objective>
