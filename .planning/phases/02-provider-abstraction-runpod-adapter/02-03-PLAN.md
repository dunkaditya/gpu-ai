**internal/provider/runpod/client.go** - GraphQL HTTP client:

Create a `Client` struct wrapping `*http.Client` with methods for each RunPod operation. Use raw `net/http` (per project convention and Claude's discretion decision — no GraphQL library).

```go
type Client struct {
    apiKey     string
    baseURL    string
    httpClient *http.Client
}
```

Core helper method `do(ctx context.Context, req graphQLRequest, result interface{}) error`:
1. JSON-marshal `graphQLRequest{Query, Variables}` into request body
2. Create `http.NewRequestWithContext` — POST to `baseURL`
3. Set headers: `Content-Type: application/json`, `Authorization: Bearer {apiKey}`
4. Execute request via `httpClient.Do()`
5. Read response body, unmarshal into `graphQLResponse{Data, Errors}`
6. **CRITICAL: Check `len(response.Errors) > 0` first** — RunPod returns HTTP 200 even on GraphQL errors (Pitfall 1 from research). Return an error containing the first GraphQL error message.
7. Unmarshal `response.Data` into the `result` parameter

Add retry wrapper `doWithRetry(ctx context.Context, op string, maxAttempts int, req graphQLRequest, result interface{}) error`:
- Retry up to `maxAttempts` times (default 3 per user decision)
- On HTTP 429: check `Retry-After` header and wait that duration (per user decision to respect rate limits)
- On HTTP 5xx or network error: exponential backoff (1s, 2s, 4s)
- On GraphQL error containing "capacity" or "insufficient" or "no available": return `provider.ErrNoCapacity` (don't retry — capacity won't appear in seconds)
- On context cancellation: return `ctx.Err()` immediately
- Non-retryable errors (4xx except 429, GraphQL validation errors): return immediately, don't retry

Define these RunPod GraphQL queries/mutations as string constants in the same file:
- `queryGPUTypes` — the `gpuTypes` query with `lowestPrice(input: {gpuCount: 1})` subquery (from research code examples)
- `mutationCreateOnDemandPod` — `podFindAndDeployOnDemand` mutation (from research)
- `mutationCreateSpotPod` — `podRentInterruptable` mutation (from research)
- `queryGetPod` — `pod(input: {podId: ...})` query (from research)
- `mutationTerminatePod` — `podTerminate(input: {podId: ...})` mutation (from research)

Create constructor `NewClient(apiKey string, opts ...ClientOption) *Client`:
- Default `baseURL`: `https://api.runpod.io/graphql`
- Default `httpClient`: `&http.Client{Timeout: 30 * time.Second}`
- `ClientOption` functional options for overriding baseURL and httpClient (needed for testing)

**internal/provider/runpod/adapter.go** - Replace the stub with full implementation:

```go
type Adapter struct {

func NewAdapter(apiKey string, opts ...ClientOption) *Adapter {

```

Implement all 5 Provider interface methods:

**`Name() string`** — returns `"runpod"`

**`ListAvailable(ctx context.Context) ([]provider.GPUOffering, error)`**:
1. Execute `queryGPUTypes` via `client.doWithRetry` with op timeout 10s (per user decision)
2. Unmarshal response into slice of RunPod GPU type structs (define internal response types)
3. For each GPU type:
   a. Call `NormalizeGPUName()` — if unmapped, log at WARN level with `slog.Warn("unmapped GPU type", "runpod_name", name)` and **skip** (per user decision)
   b. Parse pricing: `securePrice` for on-demand tier, `communitySpotPrice` for spot tier (from `lowestPrice` subquery)
   c. Determine availability: check `secureCloud` bool for on-demand, `communityCloud` bool for spot. Use `lowestPrice.stockStatus` and `lowestPrice.maxUnreservedGpuCount` for availability count
   d. Build `provider.GPUOffering` for each available tier (on-demand and/or spot), with:
      - `Provider: "runpod"`
      - `GPUType`: normalized type
      - `GPUCount: 1` (per-GPU pricing; multi-GPU is a provision-time parameter)
      - `VRAMPerGPUGB`: from RunPod's `memoryInGb`
      - `PricePerHour`: from RunPod pricing field for the tier
      - `Tier`: `TierOnDemand` or `TierSpot`
      - `Region`: from `NormalizeRegion()`
      - `DatacenterLocation`: from `NormalizeRegion()`
      - `AvailableCount`: from `lowestPrice.maxUnreservedGpuCount` (0 if unavailable)
      - `StockStatus`: from `lowestPrice.stockStatus`
4. Return the collected offerings

**`Provision(ctx context.Context, req provider.ProvisionRequest) (*provider.ProvisionResult, error)`**:
1. Apply operation timeout of 30s (per user decision)
2. Map `req.GPUType` back to RunPod GPU ID using a reverse lookup from `gpuNameMap` (find the RunPod name that maps to this GPUType). If multiple RunPod names map to the same type, pick the first found (or add a `reverseGPUNameMap` for deterministic lookup).
3. Branch on `req.Tier`:
   - `TierOnDemand`: Execute `mutationCreateOnDemandPod` with variables: `cloudType: "SECURE"`, `gpuCount`, `gpuTypeId` (RunPod name), `name` (req.InstanceID), `imageName` (req.DockerImage or default "runpod/pytorch:latest"), `startSsh: true`, `ports: "22/tcp"`, `containerDiskInGb: 40`, `env` array with SSH keys, instance ID, callback URL, internal token
   - `TierSpot`: Execute `mutationCreateSpotPod` with variables: similar to on-demand but with `bidPerGpu` set to the minimum bid price (hardcode a reasonable default or pass from request; for now use 0 to let RunPod set market price), `cloudType: "COMMUNITY"`
4. Build environment variables array: `[{key: "GPUAI_INSTANCE_ID", value: req.InstanceID}, {key: "GPUAI_CALLBACK_URL", value: req.CallbackURL}, {key: "GPUAI_INTERNAL_TOKEN", value: req.InternalToken}]`. Add SSH public keys as `SSH_PUBLIC_KEYS` (newline-joined).
5. On GraphQL error indicating no capacity: return `provider.ErrNoCapacity`
6. Unmarshal response, return `ProvisionResult` with `UpstreamID`, `CostPerHour`, `DatacenterLocation`, `Status: "creating"`, `EstimatedReadySeconds: 30`

**`GetStatus(ctx context.Context, upstreamID string) (*provider.InstanceStatus, error)`**:
1. Apply operation timeout of 10s
2. Execute `queryGetPod` with `podId: upstreamID`
3. Map RunPod `desiredStatus` to GPU.ai status:
   - "CREATED" -> "creating"
   - "RUNNING" -> "running"
   - "EXITED" -> "terminated"
   - Other -> pass through lowercased
4. Extract IP from `runtime.ports` (find port with `privatePort: 22` and `isIpPublic: true`)
5. Return `InstanceStatus` with mapped status, IP, cost, uptime, and port mappings

**`Terminate(ctx context.Context, upstreamID string) error`**:
1. Apply operation timeout of 30s
2. Execute `mutationTerminatePod` with `podId: upstreamID`
3. `podTerminate` returns null on success — check for GraphQL errors only
4. Return nil on success

Use `log/slog` for structured logging throughout (operations, retries, errors, unmapped GPUs).
