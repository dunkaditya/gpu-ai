package provider

import "errors"

var (
    ErrNoCapacity         = errors.New("no GPU capacity available")
    ErrProviderUnavailable = errors.New("provider API unavailable")

)

  <files>internal/provider/registry.go, internal/provider/registry_test.go</files>

**internal/provider/registry.go** - New file implementing the provider registry:
- `type Registry struct` with `sync.RWMutex` and `map[string]Provider`
- `func NewRegistry() *Registry` — initializes with empty map
- `func (r *Registry) Register(p Provider)` — acquires write lock, stores provider by `p.Name()`. If a provider with that name already exists, it is replaced (allows re-registration during config reload)
- `func (r *Registry) Get(name string) (Provider, bool)` — acquires read lock, returns provider and existence flag
- `func (r *Registry) All() []Provider` — acquires read lock, returns a slice copy of all registered providers. Deterministic order is NOT required (callers sort if needed)
- `func (r *Registry) Names() []string` — acquires read lock, returns sorted list of registered provider names (useful for logging at startup)
- Use `log/slog` to log registration events at Info level: `slog.Info("provider registered", "name", p.Name())`

**internal/provider/registry_test.go** - Unit tests:
- `TestRegistryRegisterAndGet`: Register a mock provider, Get by name returns it, Get unknown name returns false
- `TestRegistryAll`: Register 2 mock providers, All() returns both
- `TestRegistryNames`: Register providers, Names() returns sorted names
- `TestRegistryReRegister`: Register a provider, register another with same name, Get returns the latest one
- Use a simple mock: `type mockProvider struct { name string }` implementing `Provider` interface with stub methods that return nil/empty values. The mock only needs `Name()` to return the configured name; other methods can return zero values.
- Do NOT test concurrency with goroutines (testing race conditions is flaky). The `sync.RWMutex` usage ensures safety; just test functional behavior.
