<task type="auto">
  <name>Task 1: Create v1 schema migration SQL</name>
  <files>database/migrations/20250224_v1_schema_improvements.sql</files>
  <action>
Create a new migration file `database/migrations/20250224_v1_schema_improvements.sql` that applies ALL the following changes in a single file (the existing Python migration runner will execute it in its own transaction):

**SCHEMA-01: Rename PKs to self-documenting format**
- `organizations.id` -> `organizations.organization_id`
- `users.id` -> `users.user_id`
- `ssh_keys.id` -> `ssh_keys.ssh_key_id`
- `instances.id` -> `instances.instance_id`
- `environments.id` -> `environments.environment_id`
- `usage_records.id` -> `usage_records.usage_record_id`
- PostgreSQL automatically updates FK references when source column is renamed, but existing indexes that reference old column names must be dropped and recreated with new names.

**SCHEMA-02: Add constraints**
- `ALTER TABLE users ALTER COLUMN org_id SET NOT NULL;`
- `ALTER TABLE instances ALTER COLUMN org_id SET NOT NULL;`
- `ALTER TABLE instances ALTER COLUMN user_id SET NOT NULL;`
- Drop existing FK constraints on users.org_id, instances.org_id, instances.user_id and recreate with explicit ON DELETE behavior:
  - `users.org_id` -> ON DELETE CASCADE (org deleted = users deleted)
  - `instances.org_id` -> ON DELETE RESTRICT (can't delete org with active instances)
  - `instances.user_id` -> ON DELETE RESTRICT (can't delete user with active instances)
  - `usage_records.instance_id` -> ON DELETE RESTRICT (can't delete instance with billing records)
  - `usage_records.org_id` -> ON DELETE RESTRICT (can't delete org with billing records)
  - `environments.org_id` -> ON DELETE CASCADE (org deleted = envs deleted)
  - `environments.user_id` -> ON DELETE SET NULL (user deleted but env preserved for org)
- Add CHECK constraint on instances.status: `CHECK (status IN ('creating', 'provisioning', 'booting', 'running', 'stopping', 'terminated', 'error'))`
- Add UNIQUE constraint on instances.hostname
- Add composite unique index on `(upstream_provider, upstream_id)` -- prevents duplicate upstream references

**SCHEMA-03: Remove wg_private_key_enc**
- `ALTER TABLE instances DROP COLUMN wg_private_key_enc;`

**SCHEMA-04: Add new columns**
- `ALTER TABLE instances ADD COLUMN internal_token VARCHAR(255);` -- per-instance callback auth token
- `ALTER TABLE instances ADD COLUMN updated_at TIMESTAMPTZ DEFAULT NOW();`
- Create trigger function `update_updated_at()` that sets `NEW.updated_at = NOW()` and returns NEW
- Attach trigger to instances table: `BEFORE UPDATE ON instances FOR EACH ROW EXECUTE FUNCTION update_updated_at()`

Add a header comment explaining this is the v1 schema improvement migration. Order operations carefully: renames first, then constraint changes, then column drops, then column adds, then triggers.

Note: The migration runner (`tools/migrate.py`) handles transaction wrapping and `schema_migrations` tracking. The SQL file should NOT contain BEGIN/COMMIT.
