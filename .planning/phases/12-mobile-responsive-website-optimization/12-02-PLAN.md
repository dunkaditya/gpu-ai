**GPUAvailabilityTable.tsx filter toolbar (Issue #7 -- HIGH):**
The filter toolbar uses `flex flex-wrap items-center gap-3`. On mobile, the category chips sit between search and region/sort, wrapping awkwardly. Restructure for mobile:
- Change the outer toolbar div from `flex flex-wrap items-center gap-3 mb-6` to `flex flex-col gap-3 mb-6 sm:flex-row sm:flex-wrap sm:items-center`.
- The search input already has `w-full sm:w-64` which is correct.
- The category chips div should keep `overflow-x-auto` but add `w-full sm:w-auto sm:flex-1 sm:min-w-0` so it takes full width on mobile.
- The region select and sort button should be in a `flex items-center gap-3` wrapper that sits on its own row on mobile: wrap them in `<div className="flex items-center gap-3">` (they are likely already adjacent).

This gives a vertical stack on mobile: search (full width), category chips (full width, horizontally scrollable), region + sort (row).
