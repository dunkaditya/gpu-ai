**Footer.tsx (Issue #13):**
Change the footer grid from `md:grid-cols-5` to `sm:grid-cols-2 lg:grid-cols-5`. At the md breakpoint (768px), 5 columns is too tight. A 2-column intermediate layout at sm (640px) and full 5-column at lg (1024px) provides readable content at all widths.

**PricingWidget.tsx (Issue #5):**
The GPU tabs use `type-ui-2xs` which can get cramped with 4 tabs on narrow screens. Add responsive text sizing: on the tab buttons, add `text-[10px] sm:text-[11px]` to slightly reduce text on very narrow screens. Also add `min-w-0` to the tab buttons to allow text truncation if needed. The widget itself at `max-w-[400px]` is fine -- the tabs just need slightly smaller text.

**PricingTable.tsx:**
Read the file first. If it has a desktop-only table layout, verify it already has mobile card alternatives (per the research, it should have mobile cards). If mobile cards exist, no changes needed. If the table lacks mobile alternatives, add `hidden md:block` to the desktop table and create a mobile card layout using `md:hidden` following the InstancesTable dual-layout pattern.

Do NOT change any desktop layouts. All changes must use mobile-first Tailwind: base classes for mobile, prefixed classes (sm:, md:, lg:) for larger screens.

**About page (Issue #9):**
The company logos in team member cards have problematic sizing on mobile:
- `NovacoreLogo` uses `w-[140px] -mr-12` -- the negative margin causes overflow on narrow screens. Change to `w-[100px] sm:w-[140px] sm:-mr-12 -mr-4` so on mobile the logo is smaller with less negative margin.
- Other logos with fixed heights (`h-[56px]`, `h-[50px]`, `h-[54px]`) should add responsive sizing: `h-[40px] sm:h-[56px]` (etc.) so board member logos scale down on mobile.
- The `logoBase` class string is fine as-is.
- For the team member card grid itself, check if it already uses responsive columns. If the grid uses fixed columns at small screens, ensure `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3` (or similar) pattern.

**Free Trial page (Issue #10):**
- The main grid uses `lg:grid-cols-[1fr_520px] lg:gap-24`. On mobile this correctly stacks. But the form card likely has excessive padding. Find the form card wrapper and change padding from `p-8` (or similar) to `p-5 sm:p-8 lg:p-10` for tighter mobile padding.
- The perks grid `grid-cols-2 gap-6` in the left column -- on very narrow screens, 2 columns of perks may be tight. Change to `grid-cols-1 sm:grid-cols-2 gap-4 sm:gap-6`.

Do NOT alter desktop layouts. Only add mobile-first base classes with responsive prefixed overrides.
