  <files>frontend/src/components/cloud/InstancesTable.tsx, frontend/src/components/cloud/SSHKeyManager.tsx, frontend/src/components/cloud/GPUCard.tsx, frontend/src/components/cloud/InstanceDetail.tsx, frontend/src/components/cloud/BillingDashboard.tsx</files>
  <action>
Audit and fix all icon buttons and small interactive elements to meet the 44x44px minimum touch target (per Apple HIG / WCAG 2.5.5). The approach is to increase PADDING on small buttons without changing their visual icon size.

**InstancesTable.tsx:**
- The `CopyButton` component uses `w-7 h-7` (28x28px) -- below the 44px minimum. Change to `w-9 h-9 sm:w-7 sm:h-7` so on mobile the touch target is 36px (acceptable compromise to avoid layout disruption) and on desktop stays 28px. Alternatively, keep visual size at w-7 h-7 but add `min-h-[44px] min-w-[44px]` so the touch area is larger than the visual element. The simpler approach: change to `w-10 h-10 sm:w-7 sm:h-7` (40px on mobile, 28px on desktop).
- The edit pencil button in `EditableName` -- if it has a small click area, increase its padding similarly.
- The terminate button in the mobile cards -- ensure it has `min-h-[44px]` or is already large enough (likely is since it's a text button).

**SSHKeyManager.tsx:**
- Delete buttons for SSH keys -- ensure they have minimum 44px touch target. Look for small icon-only delete buttons and increase padding on mobile.
- The "Add Key" toggle button -- verify it's adequately sized.

**GPUCard.tsx:**
- The "Launch" button on each GPU card -- likely already adequate since it's a text button. Verify.
- Any small interactive elements (expand/collapse) -- increase padding if under 44px.

**InstanceDetail.tsx:**
- Copy buttons for SSH command and instance ID -- apply same w-10 h-10 sm:w-7 sm:h-7 pattern.
- The terminate button -- likely already large enough. Verify.

**BillingDashboard.tsx:**
- The payment toast dismiss button (the X) uses only text with `ml-4`. Add explicit sizing: `w-10 h-10 flex items-center justify-center` to make it finger-friendly.
- Credit preset buttons -- already `px-4 py-2` which gives ~40px height. Acceptable.
- Tab switcher buttons -- already `px-4 py-2`. Acceptable.

**Pattern to apply consistently:** For all icon-only buttons (copy, edit, delete, close), use:
```tsx
className="inline-flex items-center justify-center w-10 h-10 sm:w-7 sm:h-7 rounded transition-colors ..."
```
This gives 40px on mobile (close to 44px minimum, acceptable with the rounded touch area) and 28px on desktop (precise mouse targets).

Do NOT change button visual appearance significantly. The icons stay the same size -- only the touch/click area increases on mobile.
