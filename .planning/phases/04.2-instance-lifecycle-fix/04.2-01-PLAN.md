<objective>
Replace LocalhostOnly middleware on internal callback endpoints with per-instance token authentication, and add GPUCTL_PUBLIC_URL config var for callback URL construction.

Purpose: The `/internal/instances/{id}/ready` and `/internal/instances/{id}/health` endpoints are currently behind `LocalhostOnly` middleware, making them unreachable from GPU instances booting in the cloud. This plan replaces that with `InstanceTokenAuth` which validates the per-instance `internal_token` sent in the `Authorization: Bearer` header by cloud-init. It also adds the `GPUCTL_PUBLIC_URL` config var needed to construct reachable callback URLs (critical research finding: current code builds callback URL from branded hostname `gpu-xxxx.gpu.ai` which won't resolve).

Output: InstanceTokenAuth middleware, updated route registration, GPUCTL_PUBLIC_URL config, tests
</objective>

   - Add `InstanceTokenAuth` middleware function with this signature:
     ```go
     func InstanceTokenAuth(dbPool *db.Pool, next http.Handler) http.Handler
     ```
   - Implementation:
     a. Extract `instanceID` from `r.PathValue("id")`. If empty, return 400 via `writeProblem`.
        Note: `writeProblem` is in `errors.go` in the same package, so it's available.
     b. Extract Bearer token from `Authorization` header. If missing or not `Bearer ` prefixed, return 401 via `writeProblem(w, http.StatusUnauthorized, "unauthorized", "Authorization required")`.
     c. Call `dbPool.GetInstance(r.Context(), instanceID)` to look up the instance.
        - If `errors.Is(err, db.ErrNotFound)`, return 404 via `http.NotFound(w, r)`.
        - If other error, return 500 via `writeProblem`.
     d. Compare token: if `inst.InternalToken == nil || token != *inst.InternalToken`, return 403 via `writeProblem(w, http.StatusForbidden, "forbidden", "Invalid instance token")`.
     e. Otherwise call `next.ServeHTTP(w, r)`.
