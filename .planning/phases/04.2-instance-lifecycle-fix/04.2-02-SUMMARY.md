## Accomplishments
- progressStatus now polls provider.GetStatus every 5 seconds, driving the provisioning->booting state transition
- 10-minute timeout prevents goroutine leaks on stuck provisioning
- Early exit when instance concurrently terminated during polling
- SSE events published via OnStatusChange callback for both provisioning and booting transitions
- Callback URL uses GPUCTL_PUBLIC_URL when configured (falls back to branded hostname for dev)
- Full lifecycle chain now supported: creating -> provisioning -> (poll) -> booting -> (callback) -> running
