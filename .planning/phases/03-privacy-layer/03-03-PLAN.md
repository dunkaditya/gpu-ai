<objective>
Build the cloud-init template that bootstraps every GPU instance with WireGuard tunnel configuration, SSH key injection, firewall lockdown, branded hostname, provider identity scrubbing, NVIDIA driver verification, and ready callback. The template is a Go text/template embedded in the binary, rendered per-instance with validated inputs.

Purpose: This template is what makes the privacy layer work on the instance side. It configures the WireGuard tunnel back to the proxy, locks down SSH to tunnel-only access, scrubs all traces of the upstream provider, and signals readiness. Without this, instances would be bare upstream pods with no privacy guarantees.

Output: Embedded Go template, renderer with input validation, and comprehensive unit tests.
</objective>

    This replaces the old `infra/cloud-init/bootstrap.sh` (which used `{{variable}}` placeholders, not Go template syntax). The old file remains as a reference but the embedded template is the production artifact.

    The template uses Go template syntax: `{{.FieldName}}`. Use `text/template` (NOT `html/template` -- per CONTEXT.md and research, html/template would HTML-escape bash characters like `<`, `>`, `&`).

    **Template structure** (sections in order per CONTEXT.md decisions):

    ```bash
    #!/bin/bash
    # GPU.ai Instance Bootstrap Script
    # Generated by gpuctl provisioning engine -- do not edit manually
    set -euo pipefail

    # ============================================================
    # 1. WireGuard Client Installation & Configuration
    # ============================================================
    # Try kernel WireGuard first, fall back to wireguard-go userspace
    # (RunPod containers may lack kernel module or NET_ADMIN capability)
    apt-get update -qq
    apt-get install -y -qq wireguard-tools

    if ! modprobe wireguard 2>/dev/null; then
        echo "[GPU.ai] Kernel WireGuard unavailable, installing wireguard-go userspace..."
        # Download wireguard-go for userspace WireGuard
        ARCH=$(uname -m)
        case "$ARCH" in
            x86_64) WG_GO_ARCH="amd64" ;;
            aarch64) WG_GO_ARCH="arm64" ;;
            *) echo "[GPU.ai] ERROR: Unsupported architecture: $ARCH"; exit 1 ;;
        esac
        # wireguard-go binary (pre-built) -- in production this URL would be a GPU.ai-hosted artifact
        apt-get install -y -qq golang-go 2>/dev/null || true
        go install golang.zx2c4.com/wireguard-go@latest 2>/dev/null || {
            echo "[GPU.ai] WARNING: Could not install wireguard-go, WireGuard tunnel may not work"
        }
        export WG_QUICK_USERSPACE_IMPLEMENTATION=wireguard-go
        export PATH="$PATH:$(go env GOPATH)/bin"
    fi

    # Write WireGuard config
    mkdir -p /etc/wireguard
    cat > /etc/wireguard/wg0.conf << 'WGEOF'
    [Interface]
    PrivateKey = {{.InstancePrivateKey}}
    Address = {{.InstanceAddress}}

    [Peer]
    PublicKey = {{.ProxyPublicKey}}
    Endpoint = {{.ProxyEndpoint}}
    AllowedIPs = {{.AllowedIPs}}
    PersistentKeepalive = 25
    WGEOF

    chmod 600 /etc/wireguard/wg0.conf
    wg-quick up wg0 || {
        echo "[GPU.ai] ERROR: Failed to bring up WireGuard tunnel"
        # Continue anyway -- ready callback will report the failure
    }

    # ============================================================
    # 2. SSH Keys + Firewall Lockdown
    # ============================================================
    # Inject customer SSH keys
    mkdir -p /root/.ssh
    cat > /root/.ssh/authorized_keys << 'SSHEOF'
    {{.SSHAuthorizedKeys}}
    SSHEOF
    chmod 700 /root/.ssh
    chmod 600 /root/.ssh/authorized_keys

    # Remove any provider-injected SSH keys (backdoor prevention)
    # Keep only the keys we just wrote
    sed -i '/^#.*provider\|^#.*runpod\|^#.*lambda/Id' /root/.ssh/authorized_keys 2>/dev/null || true

    # Disable root password authentication
    sed -i 's/^#*PermitRootLogin.*/PermitRootLogin prohibit-password/' /etc/ssh/sshd_config
    sed -i 's/^#*PasswordAuthentication.*/PasswordAuthentication no/' /etc/ssh/sshd_config
    systemctl restart sshd 2>/dev/null || service ssh restart 2>/dev/null || true

    # Firewall: allow SSH only over WireGuard tunnel
    iptables -A INPUT -i wg0 -j ACCEPT
    iptables -A INPUT -p udp --dport 51820 -j ACCEPT  # WireGuard handshake
    iptables -A INPUT -i lo -j ACCEPT
    iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
    iptables -A INPUT -j DROP  # Block everything else (including direct SSH on public IP)

    # ============================================================
    # 3. Branded Hostname
    # ============================================================
    hostnamectl set-hostname '{{.Hostname}}'
    echo '127.0.0.1 {{.Hostname}}' >> /etc/hosts

    # Set GPU.ai branded MOTD
    cat > /etc/motd << 'MOTDEOF'

      ‚Ėą‚Ėą‚Ėą‚Ėą‚Ėą‚Ėą‚ēó ‚Ėą‚Ėą‚Ėą‚Ėą‚Ėą‚Ėą‚ēó ‚Ėą‚Ėą‚ēó   ‚Ėą‚Ėą‚ēó   ‚Ėą‚Ėą‚Ėą‚Ėą‚Ėą‚ēó ‚Ėą‚Ėą‚ēó
     ‚Ėą‚Ėą‚ēĒ‚ēź‚ēź‚ēź‚ēź‚ēĚ ‚Ėą‚Ėą‚ēĒ‚ēź‚ēź‚Ėą‚Ėą‚ēó‚Ėą‚Ėą‚ēĎ   ‚Ėą‚Ėą‚ēĎ  ‚Ėą‚Ėą‚ēĒ‚ēź‚ēź‚Ėą‚Ėą‚ēó‚Ėą‚Ėą‚ēĎ
     ‚Ėą‚Ėą‚ēĎ  ‚Ėą‚Ėą‚Ėą‚ēó‚Ėą‚Ėą‚Ėą‚Ėą‚Ėą‚Ėą‚ēĒ‚ēĚ‚Ėą‚Ėą‚ēĎ   ‚Ėą‚Ėą‚ēĎ  ‚Ėą‚Ėą‚Ėą‚Ėą‚Ėą‚Ėą‚Ėą‚ēĎ‚Ėą‚Ėą‚ēĎ
     ‚Ėą‚Ėą‚ēĎ   ‚Ėą‚Ėą‚ēĎ‚Ėą‚Ėą‚ēĒ‚ēź‚ēź‚ēź‚ēĚ ‚Ėą‚Ėą‚ēĎ   ‚Ėą‚Ėą‚ēĎ  ‚Ėą‚Ėą‚ēĒ‚ēź‚ēź‚Ėą‚Ėą‚ēĎ‚Ėą‚Ėą‚ēĎ
     ‚ēö‚Ėą‚Ėą‚Ėą‚Ėą‚Ėą‚Ėą‚ēĒ‚ēĚ‚Ėą‚Ėą‚ēĎ     ‚ēö‚Ėą‚Ėą‚Ėą‚Ėą‚Ėą‚Ėą‚ēĒ‚ēĚ  ‚Ėą‚Ėą‚ēĎ  ‚Ėą‚Ėą‚ēĎ‚Ėą‚Ėą‚ēĎ
      ‚ēö‚ēź‚ēź‚ēź‚ēź‚ēź‚ēĚ ‚ēö‚ēź‚ēĚ      ‚ēö‚ēź‚ēź‚ēź‚ēź‚ēź‚ēĚ   ‚ēö‚ēź‚ēĚ  ‚ēö‚ēź‚ēĚ‚ēö‚ēź‚ēĚ

     Welcome to GPU.ai
     Instance: {{.Hostname}}
     Support: support@gpu.ai

    MOTDEOF

    # ============================================================
    # 4. Provider Scrubbing
    # ============================================================
    # Block cloud metadata endpoint (prevents instance from discovering provider identity)
    iptables -A OUTPUT -d 169.254.169.254 -j DROP

    # Remove provider environment variables
    for var in $(env | grep -iE '^(RUNPOD_|LAMBDA_|E2E_|VAST_|COREWEAVE_)' | cut -d= -f1); do
        unset "$var" 2>/dev/null || true
    done

    # Clean provider branding from MOTD (we already overwrote /etc/motd above, but also clean alternatives)
    rm -f /etc/update-motd.d/* 2>/dev/null || true
    rm -f /etc/motd.d/* 2>/dev/null || true

    # Remove provider CLI tools
    for tool in runpodctl lambda-cloud e2ectl vastai; do
        rm -f "$(which "$tool" 2>/dev/null)" 2>/dev/null || true
    done

    # Remove provider-specific systemd services
    for svc in runpod-agent lambda-agent; do
        systemctl stop "$svc" 2>/dev/null || true
        systemctl disable "$svc" 2>/dev/null || true
    done

    # ============================================================
    # 5. NVIDIA Driver Verification
    # ============================================================
    GPU_INFO=""
    if command -v nvidia-smi &>/dev/null; then
        GPU_INFO=$(nvidia-smi --query-gpu=name,memory.total --format=csv,noheader 2>/dev/null || echo "nvidia-smi failed")
        echo "[GPU.ai] GPU detected: $GPU_INFO"
    else
        echo "[GPU.ai] WARNING: nvidia-smi not found -- GPU may not be available"
        GPU_INFO="nvidia-smi not found"
    fi

    # ============================================================
    # 6. Disable Unattended Upgrades
    # ============================================================
    systemctl stop unattended-upgrades 2>/dev/null || true
    systemctl disable unattended-upgrades 2>/dev/null || true
    apt-get remove -y unattended-upgrades 2>/dev/null || true

    # Prevent apt from auto-updating
    cat > /etc/apt/apt.conf.d/99gpu-ai-no-auto-update << 'APTEOF'
    APT::Periodic::Update-Package-Lists "0";
    APT::Periodic::Unattended-Upgrade "0";
    APT::Periodic::Download-Upgradeable-Packages "0";
    APT::Periodic::AutocleanInterval "0";
    APTEOF

    # ============================================================
    # 7. Ready Callback
    # ============================================================
    # Signal to GPU.ai API that the instance is ready
    curl -s -X POST '{{.CallbackURL}}' \
        -H 'Authorization: Bearer {{.InternalToken}}' \
        -H 'Content-Type: application/json' \
        -d "{\"gpu_info\": \"${GPU_INFO}\"}" \
        --retry 3 --retry-delay 5 --max-time 30 || {
        echo "[GPU.ai] WARNING: Ready callback failed -- instance may not show as running"
    }

    echo "[GPU.ai] Bootstrap complete"
    ```

    **Important template details:**
    - Use `text/template` syntax: `{{.FieldName}}`
    - Use heredocs with `'WGEOF'` and `'SSHEOF'` (single-quoted) to prevent bash variable expansion within the heredoc -- the Go template variables are expanded BEFORE the script runs
    - The `CallbackURL` is the full URL: `https://api.gpu.ai/internal/instances/{{.InstanceID}}/ready` -- rendered by the Go code, not by bash
    - The `AllowedIPs` field is the IPAM subnet CIDR (e.g., "10.0.0.0/16") so the instance can reach all other tunnel addresses via the proxy
    - `InstanceAddress` includes the CIDR mask (e.g., "10.0.0.5/16")
    - The provider scrubbing section handles RunPod, Lambda, E2E, Vast, CoreWeave env var prefixes
    - iptables DROP for 169.254.169.254 only (not full link-local range, per CONTEXT.md decision)

    ```go
    import (
        "bytes"
        _ "embed"
        "fmt"
        "regexp"
        "strings"
        "text/template"
    )

    //go:embed templates/bootstrap.sh.tmpl
    var bootstrapTmplStr string

    var bootstrapTmpl = template.Must(
        template.New("bootstrap").Parse(bootstrapTmplStr),
    )
    ```

    Define `BootstrapData` struct:
    ```go
    // BootstrapData holds all parameters for rendering the cloud-init bootstrap script.
    type BootstrapData struct {
        InstanceID         string // e.g., "gpu-4a7f"
        ProxyEndpoint      string // e.g., "203.0.113.1:51820"
        ProxyPublicKey     string // base64 WireGuard public key
        InstancePrivateKey string // base64 WireGuard private key (decrypted, NOT the encrypted form)
        InstanceAddress    string // e.g., "10.0.0.5/16" (includes CIDR mask)
        AllowedIPs         string // e.g., "10.0.0.0/16" (subnet CIDR for WireGuard AllowedIPs)
        SSHAuthorizedKeys  string // newline-separated SSH public keys
        InternalToken      string // per-instance callback auth token
        Hostname           string // e.g., "gpu-4a7f.gpu.ai"
        CallbackURL        string // full URL: "https://api.gpu.ai/internal/instances/{id}/ready"
    }
    ```

    Implement `ValidateBootstrapData(data BootstrapData) error`:
    - Validate InstanceID matches `^[a-zA-Z0-9-]+$` (alphanumeric + hyphens only) -- prevents shell injection
    - Validate SSHAuthorizedKeys: each line (split by `\n`) must match `^(ssh-rsa|ssh-ed25519|ecdsa-sha2-nistp(256|384|521)) [A-Za-z0-9+/=]+ .*$` OR be empty. Reject keys containing `$(`, backtick, `;`, `|`, `&` characters. This prevents shell injection via adversarial SSH key comments.
    - Validate Hostname matches `^[a-zA-Z0-9.-]+$`
    - Validate ProxyEndpoint is non-empty
    - Validate ProxyPublicKey is non-empty and looks like base64 (44 chars ending in `=`)
    - Validate InstancePrivateKey is non-empty
    - Validate InternalToken is non-empty
    - Validate InstanceAddress contains `/` (CIDR notation)
    - Return a combined error listing all validation failures

    Implement `RenderBootstrap(data BootstrapData) (string, error)`:
    1. Call `ValidateBootstrapData(data)` -- return error if validation fails
    2. Execute template: `bootstrapTmpl.Execute(&buf, data)`
    3. Return the rendered string
