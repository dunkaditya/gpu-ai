Implement these methods on `*Pool`:

1. `CreateInstance(ctx context.Context, inst *Instance) error`:
   - INSERT INTO instances with all fields
   - Use parameterized query with pgx

2. `GetInstance(ctx context.Context, instanceID string) (*Instance, error)`:
   - SELECT all columns WHERE instance_id = $1
   - Return `nil, ErrNotFound` if no rows (define `var ErrNotFound = errors.New("not found")`)

3. `GetInstanceForOrg(ctx context.Context, instanceID, orgID string) (*Instance, error)`:
   - SELECT all columns WHERE instance_id = $1 AND org_id = $2
   - Return `nil, ErrNotFound` if no rows. This enforces org ownership at the query level (AUTH-03).

4. `ListInstances(ctx context.Context, orgID string, cursor *time.Time, cursorID string, limit int) ([]Instance, error)`:
   - Base query: `SELECT ... FROM instances WHERE org_id = $1`
   - If cursor provided: `AND (created_at, instance_id) < ($cursor, $cursorID)`
   - ORDER BY created_at DESC, instance_id DESC
   - LIMIT $limit + 1 (fetch one extra to determine has_more)
   - Return the slice (caller checks len > limit to set has_more, then trims)

5. `UpdateInstanceStatus(ctx context.Context, instanceID, fromStatus, toStatus string) (bool, error)`:
   - `UPDATE instances SET status = $1 WHERE instance_id = $2 AND status = $3`
   - Use optimistic locking: check rows affected. If 0 rows, return false (state changed concurrently). If 1 row, return true.
   - This prevents race conditions (Pitfall 2 from RESEARCH.md).

6. `SetInstanceRunning(ctx context.Context, instanceID string) (bool, error)`:
   - `UPDATE instances SET status = 'running', ready_at = NOW() WHERE instance_id = $1 AND status = 'booting' RETURNING instance_id`
   - Returns false if no rows affected (already transitioned or terminated).

7. `TerminateInstance(ctx context.Context, instanceID string) (bool, error)`:
   - `UPDATE instances SET status = 'terminated', terminated_at = NOW(), billing_end = NOW() WHERE instance_id = $1 AND status NOT IN ('terminated', 'error') RETURNING instance_id`
   - Returns false if already in terminal state (idempotent). Uses NOT IN for idempotent termination (INST-06).

8. `SetInstanceError(ctx context.Context, instanceID, reason string) error`:
   - `UPDATE instances SET status = 'error', error_reason = $1 WHERE instance_id = $2`

**Provision method:**
`func (e *Engine) Provision(ctx context.Context, req ProvisionRequest) (*ProvisionResponse, error)`

Define `ProvisionRequest` (engine-level, not provider-level):
```go
type ProvisionRequest struct {
    OrgID        string
    UserID       string
    GPUType      provider.GPUType
    GPUCount     int
    Tier         provider.InstanceTier
    Region       string // optional
    Name         *string // optional user display label
    SSHKeyIDs    []string // UUIDs of ssh_keys to resolve
    MaxPricePerHour *float64 // optional price cap
}
```

Define `ProvisionResponse`:
```go
type ProvisionResponse struct {
    Instance     *provider.CustomerInstance
    PricePerHour float64
}
```

Provision steps:
1. Generate instance ID: `"gpu-" + hex(crypto/rand 4 bytes)` = 12 chars like "gpu-a1b2c3d4"
2. Generate internal token: hex(crypto/rand 16 bytes) for callback auth
3. Generate branded hostname: `instanceID + ".gpu.ai"`
4. Look up SSH keys from DB by IDs: `e.db.GetSSHKeysByIDs(ctx, req.SSHKeyIDs)` (implement this query in db/ssh_keys.go -- add this file update). If no keys found, return error "ssh keys not found".
5. Look up best provider from registry: iterate registered providers. For Phase 4, just use the first available provider (best-price selection is Phase 6). If req.Region is set, filter by region.
6. Query Redis availability cache for pricing (or if cache miss, call provider.ListAvailable). Extract price_per_hour for the requested GPU type + tier.
7. If `req.MaxPricePerHour` is set and current price exceeds it, return error with 409 status indicator.
8. Generate WireGuard key pair via `wireguard.GenerateKeyPair()`.
9. Allocate tunnel IP via IPAM (if available -- may be nil in tests).
10. Render cloud-init template using `wireguard.RenderBootstrap()`.
11. Build `provider.ProvisionRequest` with all fields.
12. Call `provider.Provision(ctx, provReq)` to create upstream instance.
13. Create `db.Instance` record with status="creating", all fields populated.
14. Call `e.db.CreateInstance(ctx, &inst)`.
15. Kick off async status progression in a goroutine: transition creating->provisioning (immediate), then the provider's async boot will trigger the ready callback for booting->running.
16. Return `ProvisionResponse` with CustomerInstance (via ToCustomer()) and PricePerHour.

NOTE on simplification: Steps 6, 8, 9, 10 involve WireGuard and Redis which may not be available in all test environments. Make these conditional:
- If `e.wgMgr == nil`, skip WG key gen, IPAM, and cloud-init render. Set WG fields to nil.
- If Redis is unavailable, use a fallback pricing lookup from the provider directly.

**Terminate method:**
`func (e *Engine) Terminate(ctx context.Context, instanceID string) error`

Steps:
1. Get instance from DB
2. Check state machine: instance must not be in terminal state (terminated, error). If already terminated, return nil (idempotent).
3. Update DB status to "stopping" using optimistic locking (`UpdateInstanceStatus`)
4. Look up provider from registry by instance.UpstreamProvider
5. Call provider.Terminate(ctx, instance.UpstreamID)
6. Call db.TerminateInstance(ctx, instanceID) to set terminated + billing_end
7. If WG manager available, remove WG peer and release IPAM address
8. Return nil on success. If provider.Terminate fails, set instance to error state with reason.
