<task type="auto">
  <name>Task 1: Fix idempotency middleware org_id type mismatch</name>
  <files>internal/api/idempotency.go</files>
  <action>
The `IdempotencyMiddleware` at lines 81, 113, 118, 155 passes `claims.OrgID` (a Clerk-format string like `"org_abc123"`) directly to `dbPool.GetIdempotencyKey`, `dbPool.CreateIdempotencyKey`, and `dbPool.CompleteIdempotencyKey`. These DB functions write to `idempotency_keys.org_id` which is `UUID NOT NULL REFERENCES organizations(organization_id)`. PostgreSQL rejects the Clerk string as invalid UUID syntax at runtime.

**Fix:** After extracting claims and before any DB calls, resolve the Clerk org ID to the internal UUID:

```go
// Resolve Clerk org ID to internal UUID for DB operations.
internalOrgID, err := dbPool.GetOrgIDByClerkOrgID(ctx, claims.OrgID)
if err != nil {
    slog.Error("idempotency: failed to resolve org ID",
        slog.String("clerk_org_id", claims.OrgID),
        slog.String("error", err.Error()),
    )
    writeProblem(w, http.StatusInternalServerError, "internal-error",
        "Failed to process request")
    return
}
```

Then replace ALL occurrences of `claims.OrgID` in the DB calls with `internalOrgID`:
- Line 81: `dbPool.GetIdempotencyKey(ctx, internalOrgID, key)` (was `claims.OrgID`)
- Line 113: `dbPool.CreateIdempotencyKey(ctx, internalOrgID, key, requestHash)` (was `claims.OrgID`)
- Line 118: `dbPool.GetIdempotencyKey(ctx, internalOrgID, key)` (was `claims.OrgID`)
- Line 155: `dbPool.CompleteIdempotencyKey(ctx, internalOrgID, key, ...)` (was `claims.OrgID`)

Add `"log/slog"` to the import block if not already imported.

Do NOT change the DB schema or the `idempotency_keys` table definition. The fix is purely in the middleware layer to resolve the Clerk ID before passing to DB.

The current code:

// 6. Clean up WireGuard peer if configured.
if e.wgMgr != nil && inst.WGPublicKey != nil && inst.WGAddress != nil {

Replace with:
```go
// 6. Clean up WireGuard peer if configured.
if e.wgMgr != nil && inst.WGPublicKey != nil && inst.WGAddress != nil {
    tunnelIP := net.ParseIP(*inst.WGAddress)
    if tunnelIP == nil {
        e.logger.Error("failed to parse WG address for cleanup",
            slog.String("instance_id", instanceID),
            slog.String("wg_address", *inst.WGAddress),
        )
    } else {
        externalPort := wireguard.PortFromTunnelIP(tunnelIP)
        if err := e.wgMgr.RemovePeer(ctx, *inst.WGPublicKey, tunnelIP, externalPort); err != nil {
            // Log but don't fail termination -- WG cleanup is best-effort.
            // The instance is already terminated in the DB and provider.
            e.logger.Error("WireGuard peer cleanup failed (best-effort)",
                slog.String("instance_id", instanceID),
                slog.String("error", err.Error()),
            )
        } else {
            e.logger.Info("WireGuard peer removed",
                slog.String("instance_id", instanceID),
                slog.String("tunnel_ip", tunnelIP.String()),
                slog.Int("external_port", externalPort),
            )
        }
    }
}
```

Add `"net"` to the import block (it is needed for `net.ParseIP` and `net.IP`). The `wireguard` package is already imported.

WireGuard cleanup is best-effort: if RemovePeer fails, log the error but do NOT return an error from Terminate. The instance is already terminated in the DB and at the provider level -- failing the entire termination because of a WG cleanup issue would leave the user in a confusing state. This matches the existing pattern in `wireguard/manager.go` where iptables cleanup is also best-effort.

Note: The IPAM module allocates addresses sequentially using `MAX(wg_address)` and has no explicit release/dealloc method. Terminated instances' `wg_address` values remain in the DB but are associated with terminated instances. Future IPAM improvements could reclaim these addresses, but for now the critical cleanup is the WireGuard peer removal (wgctrl device config) and iptables rule removal, which `RemovePeer` handles.
