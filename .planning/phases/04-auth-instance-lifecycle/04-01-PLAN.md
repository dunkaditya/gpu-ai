**internal/auth/clerk.go** -- Replace the TODO stub entirely:

- Define a context key type (unexported `claimsKey`) and a package-level `ctxKey` variable.
- Implement `ClaimsFromContext(ctx context.Context) (*Claims, bool)` that extracts Claims from context.
- Implement `ContextWithClaims(ctx context.Context, c *Claims) context.Context` that injects Claims into context.
- NOTE: Do NOT implement the Clerk middleware in this file. The Clerk SDK's `RequireHeaderAuthorization()` sets session claims via its own context. The project-specific `ClaimsFromContext` extracts from Clerk's context key using `clerk.SessionClaimsFromContext()` internally, mapping `claims.Subject` to `UserID` and `claims.ActiveOrganizationID` to `OrgID`.

Actually, simplify: `ClaimsFromContext` should call `clerk.SessionClaimsFromContext(ctx)` from the Clerk SDK, extract `Subject` (UserID) and `ActiveOrganizationID` (OrgID), and return a project `Claims` struct. This avoids maintaining two context keys. If Clerk SDK is not available (dev mode), the middleware_auth.go will handle an alternative path.
