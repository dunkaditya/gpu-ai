## Decisions Made
- All-or-nothing WG config validation: all three WG vars must be set together or all absent, preventing silent misconfiguration where only some WG vars are set
- WG initialization gated on `cfg.WGEncryptionKeyBytes != nil` (decoded bytes, not string) matching the research Pattern 1 recommendation
- IPAM subnet hardcoded to "10.0.0.0/16" matching existing IPAM tests and engine code
- `wgctrl.New()` only called inside the WG config block so dev environments without WG kernel module are never affected
